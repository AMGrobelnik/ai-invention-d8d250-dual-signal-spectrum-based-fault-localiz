## Layman Summary

Tests whether using fault-localization to pinpoint and fix errors in AI-extracted logic databases improves true/false/uncertain reasoning on the FOLIO benchmark.

## Full Summary

This artifact implements and evaluates a dual-signal spectrum-based fault localization (SBFL) pipeline for repairing LLM-extracted Prolog knowledge bases on the FOLIO natural-language reasoning benchmark.

## What was implemented

**Our method (dual_sbfl):** A full pipeline that (1) extracts a Prolog KB from natural-language premises via LLM, (2) generates oracle yes/no questions and answers to probe KB correctness, (3) converts oracle questions to Prolog goals (batched to 1 LLM call), (4) runs each goal through a meta-interpreter tracking Ochiai suspiciousness scores (direct-fault predicates) and sub-goal harvest scores (missing predicates), (5) repairs the top-K suspicious items via LLM, and (6) evaluates the conclusion using the repaired KB.

**Four baselines:** oneshot (extract KB, query conclusion directly), cot (chain-of-thought LLM reasoning in one call), selfrefine (iterative KB re-generation with oracle pass-rate feedback), flat_sbfl (binary-coverage SBFL without stratification or sub-goal harvesting — ablation of dual_sbfl).

## Dataset
FOLIO validation split (203 examples) from `tasksource/folio` on HuggingFace. Fields: premises (string), conclusion, conclusion-FOL, label (True/False/Uncertain).

## Key technical decisions
- SWI-Prolog 9.0.4 + pyswip 0.3.3 for Prolog inference
- `call_with_time_limit(3, Goal)` used as the primary timeout mechanism (SIGALRM does not reliably interrupt C extensions)
- Batch NL→Prolog conversion: all oracle questions converted in 1 LLM call instead of N calls, reducing cost ~40%
- Meta-interpreter loaded via temp .pl file (consult), uses `assertz` for coverage logging
- Clause sanitization: rejects negation-as-facts, conjunction-in-head, bare `not X`, CamelCase predicates (lowercased), and other invalid Prolog patterns from LLM output
- 256MB SWI-Prolog stack limit to handle complex KBs
- N_ORACLE_QUERIES=5, N_REPAIR_ROUNDS=1, K_REPAIR_TARGETS=3

## Results (preliminary — full 203-example run in progress at time of submission)
