## Evaluation Summary

### What this artifact provides

A full re-evaluation of the dual-signal SBFL pipeline (and four baselines) on ProofWriter-NL depth-3 (N=150 examples), producing eval_out.json with all metrics plus 7 matplotlib figures.

### Dataset
- **Main evaluation**: ProofWriter-NL RuleTaker depth-3, N=150 examples (True/False gold labels)
- **FOLIO status**: EXCLUDED — only 3 FOLIO examples were processed in the experiment artifact (preliminary, incomplete run); all main results are ProofWriter-only

### Methods evaluated
1. **dual_sbfl** (ours): LLM KB extraction → oracle yes/no query generation → LLM-based oracle coverage scoring (Ochiai suspiciousness + sub-goal harvesting) → top-K LLM repair → LLM conclusion evaluation
2. **one_shot**: Extract KB → evaluate conclusion directly
3. **cot**: Chain-of-thought single LLM call
4. **self_refine**: Iterative KB refinement with oracle pass-rate feedback
5. **flat_sbfl**: Same as dual_sbfl but without sub-goal harvesting (ablation)

### Metrics computed
- **Multi-hop deduction accuracy** per method, overall and stratified by estimated hop depth (1/2/3)
- **95% bootstrap CIs** (10,000 resamples) for pairwise accuracy differences: dual_sbfl vs. one_shot, self_refine, flat_sbfl, cot
- **Cohen's h** effect sizes for all pairwise comparisons
- **Fuzzy and strict atomic fact recall/precision** (pseudo-gold predicates from NL heuristic)
- **Hallucination rate**: fraction of KB predicates not grounded in source text (Jaccard < 0.3)
- **LLM API calls per document** (efficiency)
- **Perturbation-based oracle calibration rho** (Spearman, N=10 docs, synthetic corruptions)
- **KB structural completeness** by hop depth (extraction failure vs. repair failure vs. success)

### Fallback mode
SWI-Prolog was not available in this environment. All Prolog-dependent evaluations fell back to LLM-based equivalents: LLM oracle KB evaluation, LLM-based coverage scoring, and LLM conclusion evaluation from extracted KB text. This is explicitly noted in the output.
